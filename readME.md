RECIPE MANAGER – USER GUIDETable of Contents*Introduction**System Requirements**Installation**Configuration**Database Seeding**Starting the Application*Application Usage 7.1 Navigating the UI 7.2 Creating a Recipe7.3 Viewing &amp; Editing Recipes 7.4 Deleting a Recipe 7.5 Available HTTP EndpointsTroubleshooting--------------------------------------------Introduction The Recipe Manager is a full-stack web application for creating, viewing, editing and deleting cooking recipes. Technologies used:&nbsp;• Node.js &amp; Express (server and routing) &nbsp;• EJS (templating) &nbsp;• MongoDB with Mongoose (data storage and schema validation) &nbsp;• Bootstrap (responsive styling)System Requirements • Node.js v16 or newer • npm (included with Node.js) • MongoDB (Atlas cluster or local instance) • Git (optional, for cloning the repo)InstallationInstall dependencies: *npm install*ConfigurationIn the project root, create a file named *.env*Add these variables (replace placeholders): *MONGO_URI=mongodb+srv://&lt;username&gt;:&lt;password&gt;@cluster0.mongodb.net/recipe_db PORT=3000***Ensure .env is listed in .gitignore to keep credentials private.**Database Seeding Populate the database with sample recipes before first run: &nbsp; &nbsp; node seed.js Expected output: MongoDB connected Database seeded with 4 recipesStarting the Application&nbsp; Production mode: npm start &nbsp; Development mode (auto-reload on file changes): npm run dev Server listens on http://localhost:&lt;PORT&gt; (default PORT=3000).Application Usage7.1 Navigating the UI – Home page (/recipes) displays all recipes in card format. Screenshot placeholder: recipe-list view on desktop7.2 Creating a RecipeClick “Add Recipe” on the home page.Fill in the form: Title (min. 3 characters) Ingredients (comma-separated list) Steps (comma-separated list) Optional: Image URL (must start with http:// or https://)Submit. Client-side validation will prevent incomplete forms. Screenshot placeholder: recipe-creation form with validation7.3 Viewing &amp; Editing Recipes – To view details, click “View” on any recipe card. – On the detail page, click “Edit” to modify fields, then submit to save. Screenshot placeholder: recipe-detail view with Edit button7.4 Deleting a Recipe – On the recipe’s detail page, click “Delete.” The recipe is removed and you return to the list.7.5 Available HTTP Endpoints Method URL Description &nbsp; &nbsp; *GET /recipes List all recipes **&nbsp; &nbsp; GET /recipes/new Show recipe creation form **&nbsp; &nbsp; POST /recipes Create a new recipe **&nbsp; &nbsp; GET /recipes/:id View a single recipe **&nbsp; &nbsp; GET /recipes/:id/edit Show edit form for a recipe **&nbsp; &nbsp; PUT /recipes/:id Update an existing recipe **&nbsp; &nbsp; DELETE /recipes/:id Delete a recipe**Troubleshooting **• MongoNetworkError: **– Verify MONGO_URI is correct.**– In MongoDB Atlas, whitelist your IP under Network Access. **&nbsp;**• EJS Template Errors: **– Ensure all .ejs files are in the views/ directory. **– Confirm app.set('view engine', 'ejs') is configured in server.js.. **&nbsp;**• Module Not Found: dotenv: **– Run npm install dotenv.*
